## Step 4: Search-Exhausted Detection

### New file: `scripts/utils/status.js`
Manages `## Search Exhausted Flags` section in STATUS.md (auto-managed, do not edit manually).

Functions:
- `setSearchExhausted(nickname, searchUrl)` — flags profile in STATUS.md
- `isSearchExhausted(nickname, currentSearchUrl)` — returns true if flagged AND same URL; auto-clears if URL changed
- `clearSearchExhausted(nickname)` — removes flag

STATUS.md format added at bottom:
```
## Search Exhausted Flags
| Nickname | Search URL | Flagged Date |
```

### `connect-salenav.js` changes:
- Tracks `eligibleThisPage` (leads passing degree + already-saved filter) per page
- Tracks `consecutiveEmptyPages` — resets to 0 when any page has eligible leads
- After 10 consecutive empty pages OR `leads.length === 0`: posts Slack alert:
  ```
  🚩 LARRY FLAG -- SEARCH EXHAUSTED
  Profile: [nickname]
  Search ID: [savedSearchId]
  Result: 0 eligible leads found after 10 pages
  Action: Connection phase ending early -- new search URL needed
  ```
- Sets STATUS.md flag, ends phase
- Constant: `EMPTY_PAGE_LIMIT = 10`

### `run-profile.js` changes:
- Imports `isSearchExhausted` from `utils/status.js`
- Checks flag before connections phase at session startup
- If exhausted and URL unchanged: skips phase, adds "Connection phase skipped — search exhausted, new URL needed" to Slack summary
- If URL changed (new search set): flag auto-clears, phase runs normally
