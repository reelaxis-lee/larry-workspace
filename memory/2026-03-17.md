## InMails — Fixed & Working (2026-03-17)

### Root causes found and fixed:
1. **"Draft with AI" ghost overlay** (`_message-ghost-placeholder_jrrmou`) was intercepting clicks on the textarea — fixed by Tab-navigation from subject field into body, with `force: true` fallback
2. **Send button starts disabled** — LinkedIn only enables it after content is entered; old `fill()` method didn't trigger change events — fixed by using `.type()` with per-character delay
3. **Page load timing** — Sales Nav buttons render late; 3-6s wait was insufficient — fixed with 7-10s randomized wait + `await sleep(randomBetween(7000, 10000))`
4. **Message button not found** — `isVisible` timeout was 2000ms, bumped to 4000ms

### Key selectors confirmed working:
- Message button: `button[aria-label^="Message "]`
- Subject: `input[aria-label="Subject (required)"]`
- Body: `textarea[name="message"]` — use Tab from subject field to reach it
- Send: `button:has-text("Send")` last — polls enabled state up to 10x/500ms
